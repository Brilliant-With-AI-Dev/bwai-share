BWAI Doc — design system

The shared, light-theme frame for Brilliant With AI review documents: calm cool-slate surfaces, one blue accent, and restrained good / warn / gold signals. This page renders every component in assets/bwai-doc.css so a future author can copy the markup straight into a new artifact.

Component reference Light theme System-sans · copy-paste ready

01  Foundations

The token palette and the body type. Every color below is a --doc-* custom property — reference the token, never a raw hex value, so artifacts stay consistent if the palette shifts.

--doc-accent --doc-good --doc-warn --doc-gold

The body font is system-sans (--doc-font: -apple-system and friends), set at a calm 1.55 line height on a cool --doc-bg slate. There is no web font to load.

02  Header block

The document opener: an h1.doc title, a .lede standfirst, and a row of .meta-pills for status. Pills come in plain, .good, .warn, and .accent.

Example heading

This is the lede — one or two sentences that set up the document in plain language before any detail. Keep it readable and unhurried.

Plain pill Good pill Warn pill Accent pill

03  Section headers

Each section opens with an h2.doc containing a .num span (the accent-blue number) followed by the title, then an hr.doc rule, then a .section-sub description. Every numbered header on this page is a live example of the pattern — including this one.

04  Callouts

Two callout families. The .reco card carries a .medal glyph plus a .body, with a .gold variant for the headline recommendation. The .note block comes in plain, .info, .good, and .warn.

🔵 This is what a recommendation looks like. The lead-in is bold, and the rest explains the call in calm, plain language.
🟢 The gold variant marks the single headline takeaway — the one thing a reader should leave with. Use it sparingly, once per document.
Plain note. A neutral aside for context that doesn't rise to a recommendation.
Info note. Accent-blue framing for an explanatory detail worth highlighting.
Good note. A reassuring confirmation — something is safe or settled.
Warn note. A gentle caution about an edge or a caveat to keep in mind.

05  Constraints list

A .constraints card wraps a ul of li items, each marked with a chevron. Use inline .yes and .no spans to call out what's in and out of scope.

  • First constraint — a short rule the design holds to.
  • Scope is in for the common path and out for the rare edge.
  • Another constraint, stated plainly so a reader can hold the whole list in mind.

06  Flow chips

A .flow row of .flow-step chips joined by .flow-arrow (→). Variants: .start, .check, .end, .bad, and .ghost (dashed).

🔗 Start → ⚡ Check → ⚠️ Bad path → 💤 Ghost (dashed) → 🎯 End

07  Journey card

One .journey-card tells a single persona's story: a .journey-top header (icon, who, when), then a .journey-body holding a .ba before/now pair and a .journey-land line for where they end up.

👋

Example persona

When and why they show up.

Before What was clunky or confusing for this person previously.
Now What they experience instead, stated as a clean outcome.
→ They land on the right place for them.

08  Before / now pair

The .ba grid on its own: a .before cell (warm) and an .after cell (green), each with a .lbl label. Useful anywhere you contrast the old way with the new.

Before The previous behaviour, described so the change is easy to feel.
Now The improved behaviour, stated as a plain, positive result.
