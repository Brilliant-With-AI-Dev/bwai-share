Mirror return, recovery, and deletion

This page documents what users see when they return to Mirror, start another interview, recover a report, or delete Mirror data.

20 screenshotsDesktop viewsSynthetic data only

Behavior summary

Return actions now have separate labels.

The previous screen used “Continue” for more than one possible action. The current screen labels resume, new interview, and data deletion separately. Starting from any Mirror page now leads to the same result for the same saved account data.

01Return 02Saved information 03Reports and recovery 04Delete my data

01

Return

Mirror checks saved account data before showing a completed report, an unfinished interview, or the first-time start screen.

02

Saved information

Before another interview starts, the user can review or edit intake answers already saved in the account.

03

Reports and recovery

Completed reports remain available. If a completed interview has no report, the user can generate it again from the saved interview.

04

Delete my data

Delete my data is in Settings. It requires exact text confirmation and reports incomplete deletion as an error.
