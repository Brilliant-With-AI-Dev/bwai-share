Mirror — progressive start flow

Mockup for bapps#772. The two pre-interview checks (orientation video, mic test) become required gates, revealed one at a time on one morphing screen (not a wizard). The existing modals do the focused work; the base screen guides and gates. Click through it.

The gate is completion, not a pass. You must run the mic test and land on a result. A skip or a network block both satisfy the gate and route you into the existing text interview (micInputMode: "classic"). Nobody is locked out; nobody gets through without engaging.

Grounded vs. open. The mic flow, its states, the verbatim "voice unavailable" copy, and the passed / skipped / blocked → voice / text routing all mirror what ships today (useMicTest + MicTestModal). The block is detected when you run the test — token fetch + a Deepgram websocket with a ~10s timeout — not on arrival. Dashed-outline elements are proposed-new and need a real decision: today the start screen shows a badge only for a passed mic, nothing for skipped / blocked.
