Mirror — unified custom sign-in: pick a direction

Three on-brand treatments for the single identity-first sign-in surface at / that replaces the rejected Stytch widget, plus the 2+‑org picker and the full state set. Pick a direction — that pick unblocks build item C2. The decided flow lives in PR #460 §3; this is its visual realization, not a re-litigation of the flow.

Identity-first — no Personal/Business tabs Email magic link + Continue with Google ConsumerAuthScreen look · forest green · Newsreader/Outfit Mockups only — no repo code changed

00  How to read this

Section 01 holds the three sign-in treatments — same content and brand, different chrome — all shown in their idle resting state. Section 02 carries the org picker. Section 03 renders the complete state set on the lead variant so you can see every screen a user actually hits. These are static renders of HTML/CSS, not the live React build; the typography, color, and spacing are faithful to the proposed product.

🥇
Recommended lead: Variant A (centered card). It is the literal continuation of the existing ConsumerAuthScreen — lowest build risk for C2, identical on every viewport, and it reads as "one calm front door" rather than a marketing page. Variant B (split-panel) is the strongest pick if you want the sign-in to keep the current B2B login's branded left rail; Variant C if you want a touch more "product surface" framing without the full panel.

01  Sign-in — three treatments

  • What every variant must do (from the decided plan): one email field, an "Email me a sign-in link" primary action, "Continue with Google," and a "New here? Create an account" link to /signup.
  • No Personal / Business tabs. Identity-first discovery determines context from the email — type self-identification is unnecessary and was rejected (Matt + playbook A2). None of these variants show a type choice.
  • Password slot is a placeholder, not a decision. Variant A shows where a password field would sit. Whether it ships depends on the C1 audit (#465): if any real B2B member signs in with a password, the slot fills; otherwise it's dropped. Designed as a slot so either outcome is a one-line change.

04  What your pick decides

  • Which chrome? A (centered card), B (split-panel), or C (framed card). All three carry identical content and brand — the choice is how much surrounding weight the sign-in page should have.
  • Heading copy. Mockups use a neutral "Sign in." The old consumer screen said "Welcome back"; the B2B panel used a product headline. Flag if you want something other than "Sign in."
  • Password slot. Defaults to omitted pending #465. No action needed unless you already know passwords must stay.
  • Not in scope here: the lobby / auto-forward, the deep-link return, and account creation (/signup stays its own surface). This artifact is sign-in + org picker only.
