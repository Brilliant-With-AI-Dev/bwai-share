The Mirror wait, three ways

The letter pipeline runs 2–4 minutes. Same pipeline in all three prototypes — what changes is what the user experiences while it runs. Sims are compressed: 1 sim second ≈ 6 real seconds. Hit Replay anytime.

MOCK — Cameron fixture data, timings simulated

A — Three Observations (one extra fast AI call, in parallel)

The moment the interview ends, a small cheap-tier call generates three short observations from the conversation — fired in parallel with the pipeline, landing in ~15–25s. The user paces through them with “Next” while the letter writes behind. Ephemeral: never saved, never grounded, never shown again.

first content ~20s real synthesis — feels like the product working new prompt surface (wrongness risk before the letter) +1 LLM call per run

B — In Your Words (zero new AI calls)

No generation at all. The client already holds the full transcript — surface three striking things the user themselves said, verbatim, with a light frame (“12 minutes in, you said…”). Instant, free, and can’t be wrong — they’re the user’s own words. Selection is a simple client-side heuristic (longest substantive answers, spread across the interview).

first content instantly zero cost · zero new failure modes can’t be wrong — verbatim quotes no synthesis — reflection, not insight

C — Writer's Notebook (presentation over real pipeline events)

No new content at all — make the actual work visible. The pipeline already emits real stage events with real timings (we just instrumented them). Render them as a quiet, honest work log that ticks as things truly happen: exchanges counted, pattern search started, letter sections drafted. Nothing invented; every line maps to a real event.

first movement ~2s honest — every line is a real event zero new calls between real events, nothing moves (dead air risk)
