What's already decided

Locked-in design calls — the flow above reflects these.

  1. The prompt writes itself in front of you. No loading screen between asking and seeing — the prompt streams into the workspace as it's generated.
  2. Title, description, and prompt arrive together. The first response from the LLM includes the prompt body, the title, the description, and up to 3 sharpening questions — all in one shot, so the workspace is fully populated as soon as streaming completes.
  3. Two CTAs after the first stream, not a quiz. Once the prompt finishes generating, testers see two buttons: "Looks great — save it" (primary) or "Refine it" (secondary). Sharpening questions only appear if they click Refine.
  4. "Refine it" reveals sharpening Qs + a free-form box. Answer any of the 3 questions, type anything else in the free box, hit Apply. The prompt rewrites once. After that, all edits go through "Improve" — no more pop-up questions.
  5. Two prompt shapes, fixed. One for a one-off prompt, one for a reusable template. Testers can't edit those shapes — only what comes out of them.
  6. No user profile in this version. The system has slots for things like role, tone, audience, but we leave them empty for v1. Wired up in a later version.
  7. "+ New" reuses the welcome input. Same one sentence + reusable-template checkbox, just inside a small dialog.
  8. Reusable templates tell ChatGPT to ask first. When a tester copies a reusable template into ChatGPT, the template instructs ChatGPT to ask the user for the missing pieces before producing output.
  9. Save is a sticky bottom-right action. Always visible while scrolling, primary color, never gets buried under content. (Per Q1A.)
  10. Title and description edit inline. Click anywhere on the title or description to edit — Google-Docs style, with a subtle dotted underline on hover hinting at it. (Per Q2B.)
  11. Delete lives in the ⋯ menu. Top-right of the workspace. Keeps the prompt area clean. Library cards get a hover-revealed "⧉ Copy" affordance so testers can grab a prompt without opening it. (Per Q3.)
  12. Prompt body renders as a styled artifact. Role / Context / Task / Constraints with bold labels — not a monospace textarea. On S3.0b, it defaults to collapsed with a "Show full prompt" toggle so the workspace stays scannable.
  13. Copy comes in two flavors + a ⋯ menu. Plain text and Markdown side-by-side. ⋯ menu adds Download .txt, Download .md, Email it to me, Delete.

Open decisions

A is the recommendation. Reply by character — "1A, 2C, 3B…" — or ask for something else.

1. What metadata does the LLM return on the first response, beyond title and description?

  • A. 🥇 Title, description, tags (≤4), use-case (one-off / reusable), suggested-model hint (ChatGPT / Claude / either). Editable inline.
  • B. 🥈 A only — drop the suggested-model hint (could confuse testers).
  • C. Just title + description for v1. Defer everything else.

2. Where do tags get edited — inline only, or with a small tag-picker dialog?

  • A. 🥇 Inline — click "+ edit" next to the tag row, type or pick from a list of tags already in their library.
  • B. 🥈 Small dialog popover with autocomplete from library tags.
  • C. Not editable in v1 — LLM decides, tester lives with it.

3. Does v1 include "Email it to me" in the ⋯ menu?

  • A. 🥇 Yes — uses mailto: with subject prefilled. Zero backend needed.
  • B. 🥈 No — Download .md is already a one-click portable export, email is redundant.
  • C. Yes, with a real send (needs transactional email infra — bigger v1 ask).

4. "Start over" affordance — clears the workspace and reopens the welcome input without wiping the library. (Useful if a tester is mid-flow on a fresh prompt and wants to scrap it.)

  • A. 🥇 Skip it. The "← Library" backlink + "+ New" is one extra click and zero risk of accidental data loss.
  • B. 🥈 Add it to the ⋯ menu as "Start over" with a confirm step.
  • C. Add it as a visible "Start over" link near the title.

5. Collapsed prompt on the saved-and-settled screen — what's the default state?

  • A. 🥇 Render expanded, then animate to collapsed after ~1s so the tester sees the collapse happen and knows it can re-expand.
  • B. 🥈 Collapsed by default with a clear "▾ Show full prompt" affordance — no animation.
  • C. Always expanded — let the tester scroll. (Simpler to build.)
