What it is

Prompt Upgrader is the first Brilliant product where the user keeps the artifact across sessions — a stored, versioned prompt that an AI interviewer helps the user sharpen over time. The spike implements the v1 UX with mock state so we can react to the surface before any persistence work ships.

What to poke at

  • The dev panel (⚙ bottom-left). Flip Library / Versions / Chat / Label gen / Intake / Artifact to see every state of every route. The on-page guide explains each option.
  • Improve / Tailor / Clone. These three action surfaces on the workspace are the freshest patterns and the best places to push back.
  • One-off vs. reusable template on the New-prompt page. Required choice. Does the framing land?
  • The two layouts. Set Artifact to none in the dev panel — chat goes wide. Flip to exists — the artifact takes priority and the chat docks. Does the flip feel right?
  • Mobile. Open the workspace on a phone. Chat / Artifact tabs replace the two-pane split.

What not to react to

  • Specific prompt copy — it's placeholder.
  • The model name in metadata — placeholder.
  • Anything implying real persistence — the artifact panel is read-only by design (v1 trade per ADR-002 §1).
  • Specific colors and density — the design system pass is later.
Spike, not product. The whole point is to expose v1 UX decisions cheaply. Bidirectional canvas editing, foldering, sharing, search, and Neon-backed persistence are all explicitly deferred.
